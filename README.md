# markdown2cheatsheet

Convert Markdown notes into compact, print-ready cheatsheets with selectable layout templates and a local GUI.

[中文文档](docs/README_CN.md)

## Preview

![Output preview](outputview.jpg)

## Features

- A4 landscape cheatsheet output with selectable `4 / 5 / 6` column layouts.
- Switch color templates, fonts, font sizes, and line heights from the command line without editing `cheatsheet.css`.
- Self-contained HTML output generated by Pandoc with embedded resources.
- Local browser GUI for importing Markdown, choosing templates, previewing output, and downloading the final HTML.
- Shared conversion core used by both CLI and GUI, so behavior stays consistent.

## Requirements

| Tool | Why it is needed | Install |
| --- | --- | --- |
| [Pandoc](https://pandoc.org) | Converts Markdown to standalone HTML | macOS: `brew install pandoc` · Ubuntu: `sudo apt install pandoc` · Windows: `winget install JohnMacFarlane.Pandoc` |
| Python 3 | Runs the converter, post-processing, and local GUI | macOS/Linux: usually pre-installed · Windows: [python.org](https://python.org) |

## CLI Usage

Basic conversion:

```bash
bash md2cheatsheet.sh your_notes.md
```

Choose an output file:

```bash
bash md2cheatsheet.sh your_notes.md output.html
```

Choose a template combination:

```bash
bash md2cheatsheet.sh your_notes.md output.html \
  --columns 5 \
  --color-template forest \
  --latin-font helvetica \
  --cjk-font lxgw-wenkai \
  --code-font jetbrains-mono \
  --font-size large \
  --line-height balanced
```

List available templates:

```bash
python3 md2cheatsheet.py --list-templates
```

Run the bundled example:

```bash
bash md2cheatsheet.sh examples/test_cheatsheet.md examples/test_cheatsheet.html
```

## GUI Usage

Start the local GUI:

```bash
python3 gui_app.py
```

Double-click launchers included in the repository:

- macOS: `start.command`
- Windows: `start.bat`
- Linux: `start.sh`

Useful options:

```bash
python3 gui_app.py --port 9000 --no-browser
```

The GUI lets you:

- import a local Markdown file or paste content directly
- choose columns, color template, fonts, font size, and line height
- preview the generated cheatsheet in-browser
- download the final HTML output

## Release Build

Build a native GUI release on your current platform:

```bash
python3 -m pip install pyinstaller
python3 build_release.py
```

Output location:

- app bundle/folder: `dist/`
- release directory and archive: `release/`

Naming rule:

- `markdown2cheatsheet-gui-v1.0.0-macos-arm64.zip`
- `markdown2cheatsheet-gui-v1.0.0-windows-x64.zip`
- `markdown2cheatsheet-gui-v1.0.0-linux-x64.zip`

Release startup:

- macOS: extract the zip and launch `start.command`
- Windows: extract the zip and launch `start.bat`
- Linux: extract the zip and launch `start.sh`

Notes:

- The build is platform-native. Build on macOS for macOS, on Windows for Windows, and on Linux for Linux.
- The packager bundles the GUI assets, the default example markdown, and the stylesheet automatically.
- On macOS, the recommended entry is `start.command` inside the extracted release folder rather than directly opening the `.app` bundle.
- Version comes from the current Git tag when the checked-out commit is tagged, otherwise it falls back to a local dev version with the short commit hash.

## GitHub Actions

The repository includes a cross-platform release workflow:

- workflow file: `.github/workflows/build-release.yml`
- triggers: manual run and pushed tags matching `v*`
- output: one uploaded archive per platform under `release/`
- tag push behavior: builds all platforms and automatically creates a GitHub Release with the generated `.zip` files attached

## Template Options

### Columns

- `4`
- `5`
- `6`

### Color Templates

- `yubiquitous`: the current original project version
- `grayscale`: pure black, white, and gray
- `forest`: a muted green palette
- `sunset`: a warmer orange-red palette
- `navy`: a deep blue palette
- `plum`: a restrained purple-gray palette

### Font Choices

- Latin fonts: `arial`, `times-new-roman`, `helvetica`, `segoe-ui`, `verdana`
- Chinese fonts: `microsoft-yahei`, `pingfang-sc`, `kaiti`, `simsun`, `lxgw-wenkai`
- Code fonts: `system-code`, `consolas`, `menlo`, `hack`, `jetbrains-mono`
- Font sizes: `8`, `7.5`, `7`, `6.5`, `6`, `5.5`, `5`, `4.5`, `4`
- Line heights: `0.9`, `0.95`, `1.0`, `1.05`, `1.1`, `1.15`, `1.2`, `1.25`, `1.3`

## How It Works

```text
Markdown file
    |
    v
CLI or GUI
    - choose columns
    - choose color template
    - choose fonts
    - choose font size
    - choose line height
    |
    v
Pandoc
    - standalone HTML
    - embedded stylesheet/resources
    |
    v
postprocess.py
    - removes Pandoc table colgroup overrides
    - compresses code indentation
    - wraps body content for multi-column layout
    |
    v
Self-contained cheatsheet HTML
```

## Export to PDF

1. Open the generated `.html` file in a modern browser.
2. Press `Ctrl+P` on Windows/Linux or `Cmd+P` on macOS.
3. Set paper to `A4`, orientation to `Landscape`, and margins to `Minimum`.
4. Disable `Headers and footers`.
5. Save as PDF.
