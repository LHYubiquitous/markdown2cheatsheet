# markdown2cheatsheet

Convert Markdown notes into a compact, print-ready cheatsheet: A4 landscape, 4 columns, tiny but readable typography, and browser-based PDF export.

[中文文档](docs/README_CN.md)

## Preview

![Output preview](outputview.jpg)

## Features

- High-density A4 landscape layout with 4 balanced columns.
- Self-contained HTML output generated by Pandoc with embedded CSS/resources.
- Print-optimized typography for headings, lists, tables, blockquotes, inline code, and fenced code blocks.
- Pandoc post-processing that fixes table widths, code indentation, and multi-column wrapping.
- Works on macOS, Linux, and Windows with Git Bash or PowerShell.

## Requirements

| Tool | Why it is needed | Install |
| --- | --- | --- |
| [Pandoc](https://pandoc.org) | Converts Markdown to standalone HTML | macOS: `brew install pandoc` · Ubuntu: `sudo apt install pandoc` · Windows: `winget install JohnMacFarlane.Pandoc` |
| Python 3 | Runs the HTML post-processor | macOS/Linux: usually pre-installed · Windows: [python.org](https://python.org) |

## Quick Start

```bash
bash md2cheatsheet.sh your_notes.md
```

By default, `your_notes.md` becomes `your_notes.html`.

To choose an output path:

```bash
bash md2cheatsheet.sh your_notes.md output.html
```

Run the bundled example:

```bash
bash md2cheatsheet.sh examples/test_cheatsheet.md examples/test_cheatsheet.html
```

## Export to PDF

1. Open the generated `.html` file in a modern browser.
2. Press `Ctrl+P` on Windows/Linux or `Cmd+P` on macOS.
3. Set paper to `A4`, orientation to `Landscape`, and margins to `Minimum`.
4. Disable `Headers and footers`.
5. Save as PDF.

## Markdown Support

| Markdown | Output style |
| --- | --- |
| `# Heading 1` | Major section heading |
| `## Heading 2` | Colored subsection block |
| `### Heading 3` | Compact topic heading |
| `**bold**` / `*italic*` | Strong and emphasis styles |
| `` `inline code` `` | Compact monospace chip |
| Fenced code blocks | Syntax-highlighted, wrapped code blocks |
| `> blockquote` | Left-border callout |
| `==highlight==` | Yellow highlight through Pandoc's `mark` extension |
| Tables | Compact, content-sized table layout |

## Layout Tuning

Most layout decisions live in `cheatsheet.css`.

| Setting | Default | CSS rule |
| --- | --- | --- |
| Columns | 4 | `.content-wrapper { column-count: 4 }` |
| Page size | A4 landscape | `@page { size: A4 landscape }` |
| Page margin | 5 mm | `@page { margin: 5mm }` |
| Body font size | 6.5 pt | `html { font-size: 6.5pt }` |
| Code font size | 5.3 pt | `pre { font-size: 5.3pt }` |
| Column gap | 2.5 mm | `.content-wrapper { column-gap: 2.5mm }` |

Edit these values if you want fewer columns, larger text, or more whitespace.

## How It Works

```text
Markdown file
    |
    v
Pandoc
    - Markdown extensions
    - syntax highlighting
    - standalone HTML
    - embedded stylesheet/resources
    |
    v
postprocess.py
    - removes Pandoc table colgroup width overrides
    - compresses leading code indentation
    - wraps body content in .content-wrapper
    |
    v
Self-contained cheatsheet HTML
    |
    v
Browser print dialog -> PDF
```

## Windows Notes

The main script requires Bash. On Windows, Git Bash is the simplest option:

```bash
bash md2cheatsheet.sh your_notes.md
```

If Pandoc was installed with `winget`, the script probes the common Pandoc install directories automatically. If your shell still cannot find Pandoc, add it to `PATH`:

```bash
export PATH="$PATH:/c/Users/$USERNAME/AppData/Local/Pandoc"
```

PowerShell users can run the equivalent Pandoc command manually:

```powershell
$env:PATH += ";$env:LOCALAPPDATA\Pandoc"

pandoc your_notes.md `
  --from markdown+mark+smart+pipe_tables+fenced_code_blocks `
  --to html5 --standalone --embed-resources `
  --css cheatsheet.css --metadata title="cheatsheet" `
  --highlight-style pygments `
  --output tmp_raw.html

python postprocess.py tmp_raw.html output.html
```
